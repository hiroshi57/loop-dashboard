# DI Loop Library — エージェントループ・ダッシュボード

株式会社デジタルアイデンティティの **13サービス × 10業種** に分類した、外販品質のエージェントループ **1,005本** を、閲覧・検索・課題提案・コピーできるダッシュボードです。
ループは [loops.elorm.xyz](https://loops.elorm.xyz/loops) のキックオフプロンプト形式を踏襲しつつ、各ループに **戦略（なぜ効くか）・戦術（PDCAの回し方）** を組み込み、「回せば着実に成果が上がる」設計にしています。

典拠は [digitalidentity.co.jp/service](https://digitalidentity.co.jp/service/) と社内ナレッジベース `di-kb`（PUBLICレベル情報のみ）。クライアント名×金額・未公開情報は非掲載、数値はすべて「目標値（例）」として扱っています。

## ファイル構成

```
.
├── index.html        # ダッシュボード本体（HTML + CSS + Vanilla JS、ビルド不要）
├── loops_1000.json   # 1,005本のループデータ（生成物・ファイル名は据え置き）
├── build_loops.py    # ループ生成スクリプト（原型×業種×課題から構造化生成し自己検査）
├── DESIGN.md         # デザインシステム説明書（人間とAIが読む唯一の参照源）
├── serve.sh          # 起動スクリプト（macOS / Linux）
└── README.md
```

外部依存は CDN の Chart.js と Google Fonts（IBM Plex Sans JP / IBM Plex Mono）のみ。

## 起動方法

`index.html` は `fetch('./loops_1000.json')` でデータを読むため、**`file://` で直接開くと動作しません**。ローカルサーバー経由でアクセスしてください。

```bash
# いずれか
./serve.sh
python -m http.server 8000      # → http://localhost:8000
npx serve .
```

## 3つの分類軸と「課題からの提案」

| 軸 | 内容 |

| **サービス（13）** | 全体戦略 / SEO / コンテンツ / LLMO・Forté.AI / 運用型広告 / 広告クリエイティブ・動画 / SNS・PR / Web制作・LP / CRO / マーケDX / WEB解析 / リサーチ / CMO代行・インハウス |
| **業種（10）** | 業種横断 / 金融・保険・証券 / 不動産・建築 / 医療・クリニック / 旅行・ホテル・観光 / 自動車 / EC・小売 / BtoB・SaaS / 人材・教育 / 公共・団体 |
| **課題（15）** | CVが伸びない / 獲得単価が高い / 流入が足りない / 順位が下がった / AIに見つけられない / ブランドが弱い / LTVが低い / 解約が多い / リードの質 / データで判断できない / クリエイティブが当たらない / コンプラ不安 / 戦略がない / 内製化したい / 顧客が分からない |

- 上部タブで **「サービス別 / 業種別 / 課題で絞る」** を切替。チャートも軸に連動。
- **「課題から探す」** セクションで課題を選ぶと、それに効くループが**複数**提案されます（1ループは複数課題に紐づく）。

## 各ループに含まれる情報

戦略コンサルのフレーム（`isshi-inc.jp/chisoku` 参照）を取り込み、以下を全ループに付与：

- **戦略**: 業種の戦略観（限られた資源で勝つ選択の指針）＋このサービスの勝ち筋
- **KBF / KSF**: 顧客の決め手と、勝ち筋
- **ASIS → TOBE**: 現状と目標のギャップ
- **ループ定義**: Goal / 背景・前提 / Max iterations / Between iterations run / Exit when / Step 1
- **戦術（PDCA）**: 各イテレーションで回す Plan→Do→Check→Action の具体手順
- **成果物 / KPI / ガードレール**: 業種別の規制対応（薬機法・金商法・景表法・アクセシビリティ等）
- **Loopを構成する6つのコアモジュール**: Automations（ハートビート）/ Worktrees（防護壁）/ Skills（記憶チップ）/ Sub-agents（牽制メカニズム）/ Connectors（腕）/ Memory（過小評価されている部品）。単発のCronではなく1つの完全フローとして連動させる。Skills には [Hermes Agent Skills Hub（agentskills.io）](https://hermes-agent.nousresearch.com/docs/skills/) の実在スキルを、サービス×業種×課題の関連度で割り当てる
- **キックオフプロンプト**: 上記を束ねた、貼ってすぐ使える全文（Copyボタンでクリップボードへ）

## 機能

- KPIサマリー、サービス/業種の横棒チャート（タブ切替・クリック絞り込み）、カテゴリ構成ドーナツ、トリガー内訳
- フリーワード検索（タイトル / Goal / 戦略 / 背景 / タグ / 業種 / サービス）
- 3軸タブ＋課題提案＋カテゴリ/トリガーチップの併用フィルタ、適用中フィルタのピル表示・個別/一括解除
- ループカード（サービス/業種/カテゴリ/トリガーのバッジ、戦略1行、イテレーションリング、KPI・課題タグ）
- 詳細モーダル（戦略→ループ定義→戦術→成果物/KPI/ガードレール→課題→KICKOFF PROMPT）
- 24件/ページのページネーション、ダークモード（記憶）、レスポンシブ、アクセシビリティ対応

## データの再生成

```bash
python build_loops.py
```

`build_loops.py` は **109の固有原型**（サービス専用＋クロスセル/LTV等の拡張）を、各原型の「適合業種」にだけ展開し、業種ごとに goal/step1/KPI/ガードレール/戦略を差し替えて **約1,000本** を生成します。
「強化版」のような中身が同一の水増しは行いません。生成後に **空フィールド0・未置換プレースホルダ0・id/title重複0・カバレッジ集計** を自己検査します。

## デザイン（DESIGN.md）

UIのルール（カラートークン・タイポグラフィ・余白・コンポーネント・Do/Don'ts・エージェント向けガイド）は `DESIGN.md` に明文化しています（Google Stitch の DESIGN.md / awesome-design-md の9セクション構成に準拠）。
UIを変更する際は **DESIGN.md の値のみを参照源** とし、変更したら DESIGN.md も同時に更新してください。

## 検証済みの完了条件

- 1,005件すべてに戦略・戦術・KPI・ガードレール・課題タグが付与され、空フィールド・未置換・重複なし
- サービス13 / 業種10 / 課題15 すべてに複数ループが紐づく（課題からの複数提案が成立）
- ローカルサーバーで index.html / loops_1000.json が 200 で配信、全色キー網羅、JS要素ID・関数・括弧整合
- Copy でキックオフプロンプトがクリップボードにコピー

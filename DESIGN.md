# DESIGN.md — DI Loop Library ダッシュボード デザインシステム

> これは「人間とAIの両方が読めるデザインの説明書」です（Google Stitch の DESIGN.md の考え方に準拠）。
> UIを変更・拡張する際は、必ずこのルールに従って一貫性を保つこと。曖昧な「なんとなく」の指示を避け、
> ここに明文化された値・原則を唯一の参照源（Single Source of Truth）とする。
> 実装の正本は `index.html` の `:root` / `html[data-theme="dark"]` CSS変数。本書と実装が食い違う場合は本書を更新する。

---

## 1. デザインの方向性（Tone & Principle）

- **性格**: プロフェッショナル / 信頼できる / データドリブン。外販に耐える「よく考えられた」印象。
- **比喩**: SaaSの管理コンソール × 戦略コンサルの提案資料。装飾過多にしない。
- **AIっぽさの排除**: 過剰なグラデーション、絵文字の多用、紫一色のグラデ、意味のない影を避ける。
  情報設計（階層・余白・整列）で「整って見える」状態をつくる。
- **原則**:
  1. **一貫性** — 同じ意味には同じ色・同じ角丸・同じ余白を使う。
  2. **明確性** — ラベルは日本語で具体的に。1画面1主題。
  3. **可搬性** — 依存はCDN（Chart.js / Google Fonts）のみ。ビルド不要の単一HTML。
  4. **アクセシビリティ** — フォーカスリング・コントラスト・reduced-motion を担保。

---

## 2. カラートークン（Light / Dark）

CSS変数で定義し、ハードコードしない。ダークは `html[data-theme="dark"]` で上書き。

| トークン | 用途 | Light | Dark |
|---|---|---|---|
| `--bg` | ページ背景 | `#F2F4F8` | `#0D1020` |
| `--surface` | カード/パネル面 | `#FFFFFF` | `#161A2E` |
| `--surface2` | サブ面（入力・バー等） | `#F7F8FB` | `#1C2138` |
| `--ink` | 主要テキスト | `#171B2E` | `#E9EBF5` |
| `--sub` | サブテキスト | `#5B6175` | `#A6ACC4` |
| `--faint` | 補助テキスト/枠 | `#8A90A6` | `#6E7490` |
| `--line` | 罫線（薄） | `#E2E5EE` | `#272D48` |
| `--line2` | 罫線（濃）/枠 | `#D2D6E4` | `#343B5C` |
| `--primary` | 主アクセント | `#2B4BD8` | `#7B93FF` |
| `--primary-ink` | primary上の文字 | `#FFFFFF` | `#0D1020` |
| `--primary-soft` | primary淡背景 | `#E7EBFB` | `#23294A` |
| `--ok` | 成功/intervalトリガー | `#0E8A5F` | `#3FCE9A` |

**使い分け**: 文字色は `--ink > --sub > --faint` の3階層のみ。背景は `--bg / --surface / --surface2` の3層のみ。

### 軸カラー（分類の識別色）
- **サービス（13）**: `SVC_COLORS` に各サービスキーの固有色を定義（既存）。
- **業種（10）**: `IND_COLORS` に各業種キーの固有色を定義（業種バッジ・業種チャートで使用）。
- **カテゴリ（6）**: `CAT_COLORS`。バッジ左ボーダー・ドーナツに使用。
- **課題（pain）**: 強い色は付けず、`--primary-soft` / `--sub` 系で「タグ」として表現する。
- バッジ文字色: サービス/業種の濃色背景には白文字（`#fff`）。

## 3. タイポグラフィ

- **本文フォント**: `"IBM Plex Sans JP", system-ui, sans-serif`
- **等幅フォント**: `"IBM Plex Mono", monospace`（数値・ID・KPI・eyebrow・プロンプト）
- **基準サイズ**: body `14.5px` / `line-height:1.65`
- **スケール**:
  | 用途 | size | weight |
  |---|---|---|
  | KPI数値 | 30px | 600 |
  | モーダル見出し/サービス詳細h3 | 16px | 700 |
  | ブランドh1 | 15px | 700 |
  | カードタイトルh4 | 14px | 600 |
  | パネルh3 | 13px | 600 |
  | 本文/Goal等 | 12.5〜13.5px | 400 |
  | バッジ | 10.5px | 600 |
  | eyebrow / f-label | 11px mono | 600（letter-spacing広め・大文字） |
- 数値・KPI・ID・コードは必ず `.mono`（等幅）にする。

---

## 4. スペーシング・形状

- **角丸**: `--radius:14px`（カード/パネル）。入力・チップ・ボタンは `9〜10px`。ピル/丸ボタンは `999px`。モーダルは `18px`。
- **コンテナ幅**: `.wrap { max-width:1240px; padding:0 20px }`
- **グリッド**: カード `repeat(3,1fr)` / gap `14px`。KPI `repeat(4,1fr)` / gap `12px`。
- **影**: `--shadow`（薄い2段影）。多用しない。ホバーで `translateY(-2px)` 程度。
- **セクション見出し（eyebrow）**: `NN / TITLE` 形式の mono大文字 + 右に伸びる罫線。

---

## 5. コンポーネント規約

- **カード（.card）**: バッジ群（サービス/業種/カテゴリ/トリガー）→ タイトル → goal(2行省略) → 戦略1行 → イテレーションリング+exit → KPI/課題タグ → フッター（View / Copy）。`fadeUp` で出現。
- **バッジ（.badge）**: `svc`=サービス色背景・白文字 / `ind`=業種色背景・白文字 / `cat`=左ボーダー色+薄背景 / `trg`=mono枠線（interval は `--ok`）。
- **チップ（.chip）**: フィルタ選択。選択時 `.on` = primary背景。
- **タブ（.axis-tabs）**: サービス別 / 業種別 / 課題別 の切替。選択タブは primary下線または primary背景。
- **ボタン（.btn）**: 既定=枠線。`.primary`=primary背景。Copy成功時 `.copied`=`--ok`。
- **モーダル（.modal）**: 戦略（strategy/KBF・KSF/ASIS-TOBE）→ ループ定義（Goal/背景/Max/Check/Exit/Step1）→ 戦術（tactics）→ 成果物/KPI/ガードレール → KICKOFF PROMPT（mono・コピー可）。
- **トースト（.toast）**: 下中央。コピー完了などの一時通知。

---

## 6. モーション・アクセシビリティ

- トランジション: `.15s〜.25s`。テーマ切替は `.25s`。
- `:focus-visible` に `2px solid var(--primary)` のアウトライン。
- `@media (prefers-reduced-motion:reduce)` で全アニメーション/トランジションを無効化。
- レスポンシブ: `980px` でカード2列・チャート1列・KPI2列、`620px` でカード1列。

---

## 7. 文言・トーン

- ラベルは日本語で具体的に（「該当 N 件」「すべて解除」等）。
- 機能名の英字（Copy / View / KICKOFF PROMPT）は mono で短く。
- 数値は「目標値（例）」と分かる文脈で示し、断定的な成果保証表現を避ける（特に金融・医療）。

---

## 8. レスポンシブ動作

| ブレークポイント | 挙動 |
|---|---|
| 〜620px | カード1列 / KPI2列 / `.hd-count`非表示 / f-labelは100%幅 |
| 〜980px | カード2列 / チャート1列 / KPI2列 |
| 980px〜 | カード3列 / KPI4列 / チャート2カラム |

- レイアウトは `.wrap`（max 1240px）中央寄せ。グリッドは `grid-template-columns` のメディアクエリで段階変更する。
- タブ・フィルタ行は `flex-wrap:wrap` で折り返す。

## 9. エージェント向けプロンプトガイド（AIがUIを生成・変更するときの指示）

> このダッシュボードのUIを生成・変更するAIエージェント（Claude Code / Cursor 等）は、**本DESIGN.mdの値のみを唯一の参照源**として使うこと。記載のない色・サイズ・角丸を勝手に発明しない。

- **色**: §2のCSS変数トークンを使う。`#xxxxxx` 直書きは「軸カラー定義（SVC/IND/CAT_COLORS）」と「CSS変数定義そのもの」以外で増やさない。
- **角丸**: カード/パネル=14px、入力/チップ/ボタン=9〜10px、ピル=999px、モーダル=18px。フラット過多にもせず、丸すぎにもしない。
- **影**: `--shadow` のみ。新規 `box-shadow` を発明しない。ホバーは `translateY` で表現。
- **フォント**: 本文=IBM Plex Sans JP、数値/コード/ラベル=IBM Plex Mono。サイズは§3スケールから選ぶ。
- **禁止事項（Don'ts）**: 意味のないグラデーション / 絵文字の多用 / 紫一色のAIっぽい配色 / 複数グレー値の無秩序な混在 / 影の多用 / 中央寄せ多用による締まりのなさ。
- **やること（Do's）**: 余白と整列で階層を作る / 同じ意味に同じトークン / 日本語の具体的ラベル / mono を数値に徹底 / ダーク対応をトークンで担保。
- **このプロジェクト固有**: 3つの分類軸（サービス/業種/課題）はタブで切替。課題（pain）から複数ループを提案する導線を持つ。ループ詳細は「戦略→ループ定義→戦術→成果物/KPI/ガードレール→プロンプト」の順で見せる。

---

## 10. 変更時のルール

1. 新しい色が必要なら、まずトークンで表現できないか検討する。直書き（`#xxxxxx`）はチャート/軸カラー定義以外で増やさない。
2. 新コンポーネントは本書の角丸・余白・影・タイポのスケールに合わせる。
3. UIを変えたら本書（DESIGN.md）も同時に更新し、実装と一致させる（実装＝`index.html`、人間プレビュー＝ブラウザ表示）。
